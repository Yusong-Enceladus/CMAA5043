What makes it work
Three pillars

One model, three honest claims.

01 · Living manual

Model and manual share one source.

Voice, tap, and structural edits all flow through a single mutation engine. The 3D view, the printed manual, and the change log update from the same data — never out of sync.

02 · Declarative bricks

Every brick is data.

Models are stored as structured brick data. Inventory counts, manual steps, and 3D rendering all read from the same source — so adding a new model means adding one JSON file, not rewriting screens.

03 · Single web page

Runs from a URL.

Deploys to Cloudflare's edge as a single-page web app. No app store, no install, no parent setup. Open the link, point a webcam, build something. It works on a Chromebook in a school.
