Why it ships
Six design decisions

The moat is the stack.

📖
01 · LIVING MANUAL

The manual rewrites itself

Voice, tap, and structural mods all flow through one mutation engine. The 3D viewer and the printed manual update from the same source — never out of sync.

⚡
02 · LOCAL FIRST

Deterministic on-device parser

Every voice command and tap edit is parsed locally — no LLM round-trip. Zero stalls, zero offline failure, zero cost per session.

🧩
03 · DATA BRICKS

Every brick is JSON

Models are fully declarative. The same 3D pipeline renders the Robot Dog, a Robot Car, a Dino Bot, or whatever the child asks for tomorrow.

🎯
04 · REGION AWARE

"Make the head purple"

The parser knows aliases — head = snout = face — so the natural words a six-year-old uses just work. Voice scopes change to a region; tap scopes to one brick.

🪶
05 · ZERO INSTALL

Single-page web app

A URL, a webcam, a microphone. Works on a Chromebook, iPad, or any laptop. No App Store, no parent setup, deploys to Cloudflare's edge.

👶
06 · SIX-YEAR UX

Tuned for small attention spans

Big type, warm palette, always-visible primary action, no hidden menus. Every screen is single-finger reachable; every error is a friendly Buddy bubble.
